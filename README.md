1. Структура на проекта
Проектът е организиран на модулен принцип, разделяйки изходния сорс код на приложението от инфраструктурната конфигурация на контейнерите. Това позволява пълна преносимост и независимост на жизнения цикъл на услугите.

Plaintext
C:\Docker/
├── docker-compose.yml       # Главен оркестрационен файл за услугите
├── Dockerfile               # Инструкции за изграждане на потребителския PHP образ
├── database.sql             # Почистен SQL скрипт за инициализиране на таблиците
└── src/                     # Директория със сорс кода на PHP приложението
    ├── config.php           # Конфигурация за връзка с базата данни (PDO)
    ├── index.php            # Главна страница на уеб сайта
    └── [други PHP файлове]  # login.php, posts.php и останалата бизнес логика
2. Компоненти на системата и тяхната работа
Системата е изградена от два изолирани компонента (контейнери), които работят в синхрон:

А. Уеб Сървър (web)
Технологичен стек: PHP 8.2 + Apache.

Роля: Приема входящите HTTP заявки от браузъра на порт 8080. Изпълнява PHP скриптовете от споделената папка src/ и връща генерираното съдържание към потребителя.

Dockerfile: Тъй като официалният базов образ на PHP няма вградени драйвери за връзка с MySQL, се използва наш Dockerfile. Чрез него се изпълнява операцията:
RUN docker-php-ext-install pdo pdo_mysql
Тя компилира нужните драйвери, без които PHP кодът не може да комуникира с базата.

Б. База Данни (db)
Технологичен стек: MySQL 8.0.

Роля: Съхранява дълготрайно данните на приложението. Обслужва заявките от PHP приложението на вътрешния за мрежата порт 3306.

Устойчивост на данните (Volumes): Конфигуриран е Docker том (mysql_data), който гарантира, че информацията в базата няма да се изтрие при спиране или рестартиране на контейнера.

3. Комуникация и мрежова свързаност между услугите
Комуникацията е реализирана чрез вградената мрежова изолация на Docker Compose:

DNS Резолюция вместо localhost: Вътре в Docker средата думата localhost се отнася за самия контейнер, в който се намира кодът. Затова уеб сървърът не може да намери базата на localhost. Docker Compose автоматично създава вътрешна мрежа и DNS, благодарение на които PHP се свързва с MySQL, използвайки името на услугата като хост: host=db.

Изолация на портовете: * Уеб сървърът пренасочва вътрешния порт 80 към външния порт 8080, за да бъде достъпен от браузъра на адрес http://localhost:8080.

MySQL сървърът слуша на порт 3306 вътре в Docker мрежата за връзка с PHP, но е изложен навън на порт 3307 за администраторски нужди (например връзка през MySQL Workbench от компютъра), избягвайки конфликти с локални инсталации като XAMPP.

Оркестрация (depends_on): Указано е, че услугата web зависи от db. Това гарантира правилна последователност при стартиране — уеб сървърът изчаква първоначалното вдигане на базата данни.

4. Изграждане, стартиране и първоначално импортиране
А. Изграждане и стартиране
Всички услуги се изграждат и пускат едновременно чрез терминала в главната папка с една единствена команда:

Bash
docker compose up -d --build
Флагът -d пуска контейнерите в скрит режим (на заден план), а --build принуждава Docker да компилира Dockerfile файла наново.

Б. Автоматично импортиране на базата данни
В настройките на MySQL е заложено мапиране на файла: ./database.sql:/docker-entrypoint-initdb.d/init.sql.
Когато контейнерът на MySQL се стартира за първи път и открие празна база данни, той автоматично изпълнява този SQL скрипт. По този начин схемата на базата данни се импортира автоматично и създава таблиците users и posts без нужда от ръчна намеса.

В. Полезни команди за управление
Проверка на статуса: docker compose ps (Уверява ни, че и двата контейнера имат статус Running).

Преглед на логове: docker compose logs db (Използва се за проследяване на успешното стартиране и импортиране на SQL файла).

Пълно зануляване: docker compose down --volumes (Спира проекта и изтрива базата от данни, което позволява следващото стартиране да я изгради напълно начисто от SQL файла).
